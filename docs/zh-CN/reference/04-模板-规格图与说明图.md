# 模板：规格图 / 使用说明图

## 1) 为什么必须模板化出字
规格图与说明图对“文字一致性”要求极高，而纯 AI 出字容易出现：
- 数值错误/单位错/拼写错
- 字体不稳定、字号漂移
- 同一产品多张图不一致

因此建议：背景/场景可以 AI 生成，但文字层用模板渲染（可复现、可审计）。

## 2) 通用画布与安全区（建议）
- 画布：`2000x2000`（默认）
- 安全区：四周留边 ≥ `120px`
- 文案区：固定在下方或侧边（避免压到产品主体）

## 3) 规格图（Specs）模板字段（示例）
> 文本“内容”必须保持一致：数值/单位/含义不变。

- 标题：`<Product Name>`（英文）
- 尺寸：`<L> x <W> x <H>`（单位统一，如 cm / in，按业务决定）
- 关键参数列表（3–8 条）：`<Spec Key>: <Spec Value>`
- 注释（可选）：合规/测量误差说明（英文固定句式）

## 4) 说明图（How-to）模板字段（示例）
> “含义一致”，可轻微改写英文措辞，但步骤/注意点不能变更。

- 标题：`How to Use` / `How It Works`（可按品类）
- 步骤：`Step 1/2/3...`（每步 6–12 个英文单词）
- 注意事项：`Tips / Notice`（2–4 条）

## 5) 字体与字号策略（满足“必须不同/必须一致”）
- 规格图：字体必须不同于供应商图（建立“字体白名单库”，生成时随机或按风格包指定）。
- 说明图：字体必须不同于供应商图（同上）。
- 个性化文字（如定制祝福语）：内容不同，但字体与字号一致
  - 做法：同一产品同一订单的“个性化文字层”固定一个字体与字号；只替换文字内容。

## 6) 背景变化策略（保证“背景必须不同”）
- 规格/说明图背景：可用同类简洁背景（渐变/纹理/轻场景），但必须与供应商图不同。
- 可复用做法：为每个风格包提供 3–5 个背景变体（颜色/纹理/构图不同）。

## 7) 建议的“字段源”（Source of Truth）
> 用于自动化与审计：让每张规格/说明图都能追溯其文字来源。

- `spec_source`：来自供应商规格图 OCR / 来自经理表单 / 来自产品数据库
- `howto_source`：来自供应商说明图 OCR / 来自经理表单
- `language_policy`：`input: EN|RU`, `output_text: EN`
