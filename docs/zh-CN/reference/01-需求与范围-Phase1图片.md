# Phase 1：图片｜需求与范围

## 1. 项目目标（Project Goal）
为网店建立 AI 驱动的商品图片生成工作流：以供应商产品照片为输入，自动生成“可商用、品牌化、与供应商图明显不同、但产品本体保持一致”的图片资产。

## 2. 范围（Scope）
### 2.1 本阶段交付（Phase 1：图片）
- 图像生成/再创作工作流（Image Re-composition Workflow）。
- 支持将自定义内容（文字/照片/Logo/图形）融入生成图中。
- 支持生成：主图/场景图、尺寸规格图、使用说明/How-to 图。

### 2.2 不在本阶段（Phase 2：视频）
- 任何视频生成、剪辑、动效、视频模板化、视频批处理能力。

## 3. 输入输出定义
### 3.1 输入（Inputs）
- 供应商产品图片（必需）
  - 产品 1–4：供应商图 +（可能有）描述 + 期望输出示例
  - 产品 5–8：仅供应商图
- 个性化定制信息（按订单类型变化）
  - 自定义照片（用户上传）
  - 自定义文字（用户提供）
  - 自定义 Logo / 图形（用户提供）
- 语言要求
  - 经理输入 AI 指令：英语 / 俄语
  - 生成图片上的文字：必须为英语

### 3.2 输出（Outputs）
对每个产品，至少包含以下类别（数量可按上架模板扩展）：
- 主图/场景图（Product Showcase Images）
- 产品尺寸规格图（Size / Specs Images）
- 如何使用/产品说明图（How-to / Instruction Images）

## 4. 核心约束（Non-negotiable Constraints）
### 4.1 必须保持相同（Must Stay the Same）
- 产品本体保持一致：外观、结构、颜色、形状不允许改变；真实感与比例合理。
- 规格图：文字内容（数值/尺寸/单位/含义）保持一致（字体必须变化、背景必须变化）。
- 说明图：文字含义保持一致（可轻微改写）（字体必须变化、背景必须变化）。

### 4.2 必须不同（Must Be Different）
- 背景必须不同：可同类风格，但不得同背景/同构图复刻。
- 个性化内容必须“不同但相似（Different but Similar）”
  - 自定义照片：不同照片，但风格/语义相近
  - 自定义文字：文案不同，但字体与字号一致
  - 自定义图形/Logo：不同图形，但风格/轮廓/类别相似

## 5. 视觉规则矩阵（对齐甲方原文）
| 模块 | 目标 | 保持 | 改变 |
| --- | --- | --- | --- |
| 产品本体 | 100% 一致 | 外观/比例/材质观感 | 不允许改变 |
| 背景 | 明显不同 | 可保持同类风格 | 场景/光线/构图/道具/色调变化 |
| 个性化照片 | 不同但相似 | 主题/风格一致 | 换成另一张相似照片 |
| 个性化文字 | 不同但一致 | 字体与字号一致 | 文案内容不同 |
| 个性化图形/Logo | 不同但相似 | 视觉类别/风格一致 | 图形本身不同 |
| 尺寸规格图 | 信息一致 | 文本内容一致 | 字体不同 + 背景不同 |
| 使用说明图 | 含义一致 | 含义/步骤一致 | 措辞轻改 + 字体不同 + 背景不同 |

## 6. 工作流设计要求（面向非技术经理）
- 最优：单次上传供应商图 → 自动生成全部目标图片。
- 若无法完全自动化：采用最少步骤（上传 → 选择模板/风格 → 一键生成）。
- 可扩展：支持仅凭图片也能生成合格输出；描述仅作辅助。
- 多语言：经理可用英语或俄语输入；所有出图文案统一英文。

## 7. 质量验收标准（Acceptance Criteria）
### 7.1 产品一致性（Product Lock）
- 对比供应商图：应可判定为“同一款产品”。
- 不允许：形态变化、关键结构缺失、图案错位、比例失真。

### 7.2 差异化（Non-duplication）
- 对比供应商图：背景/构图/视觉语言必须明显不同。
- 不允许：直接抠图 + 同背景复刻、同角度同摆放的近似复制。

### 7.3 个性化规则符合性
- 照片/图形：满足“不同但相似”。
- 文字：内容不同，但字体/字号一致。
- 输出文字为英文。

### 7.4 规格图/说明图规则符合性
- 规格图：文本内容一致；字体不同；背景不同。
- 说明图：含义一致；措辞轻改；字体不同；背景不同。

## 8. 风险点（必须写清）
- “产品不变”与“背景显著不同”冲突：需要可靠的产品保护/锁定策略。
- 仅图片输入泛化难：品类越多越依赖模板化/规则化。
- 字体/字号稳定性：需明确字体来源与渲染方式（建议模板渲染，而非纯 AI 出字）。
- 合规目标是“不同但相似”：需明确衡量方式（人工规则/阈值/日志留痕）。
