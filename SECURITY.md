# Security Policy

## Supported scope

This project is documentation/template-centric. Security issues usually involve:

- Accidental leakage of API keys or credentials in examples
- Inclusion of non-commercially licensed assets
- Unsafe automation instructions that could expose private data

## Reporting

Please open a private security report through GitHub Security Advisories when possible.
If unavailable, open an issue without sensitive details and request private follow-up.

## Best practices

- Never commit secrets.
- Use redacted examples.
- Validate commercial licenses for fonts/images/icons.
