# Contributing Guide

Thanks for contributing.

## Scope

This repository focuses on reusable documentation, templates, and workflow standards for e-commerce image generation.

## How to contribute

1. Open an issue describing the change.
2. Fork and create a feature branch.
3. Keep changes minimal and focused.
4. Update related templates/checklists when changing workflow rules.
5. Submit a pull request with:
   - what changed
   - why it changed
   - how to validate

## Content conventions

- Prefer clear operational language over abstract theory.
- Keep QA criteria testable.
- Use English for globally shared docs under `docs/en`.
- Use UTF-8 and Markdown.

## Pull request checklist

- [ ] Links are valid
- [ ] New fields are reflected in templates
- [ ] QA checklist updated if acceptance criteria changed
- [ ] No private or sensitive data included
